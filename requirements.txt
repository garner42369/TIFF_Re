# 首先安装系统级别的 GDAL C++ 库（如 libgdal-dev，版本通常是 3.4.x 或 3.6.x）。
# 
# 为了让 Python 的 GDAL 库与底层 C++ 库的 API 完全对齐，必须：
# 1. 绝对不能在这里写死具体版本号 (如 gdal==3.6.4)。
# 2. 如果写死版本号，且与系统的版本不一致，部署时会直接报 "gdal-config not found" 或 "SWIG version error" 导致构建崩溃。
# 3. 留空版本号 `gdal` 会让 pip 安装时自动调用 `gdal-config` 去嗅探系统版本，从而编译出 100% 兼容的 Python Wheel。
# ---------------------------------------------------------
